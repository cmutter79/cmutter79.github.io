Standing It Up

Step 1: Create the GitHub Repo

I created a repository named exactly cmutter79.github.io. This is the magic name GitHub uses for user pages. Anything I push to this repo gets published automatically at https://cmutter79.github.io.

Step 2: Clone Locally and Add My Site

Using VS Code, I cloned the repo and dropped in my Jekyll theme files. From there, I started tweaking:

  • Updated the _config.yml file with my site name, description, and author info
  • Set baseurl: "" and url: "https://cmutter79.github.io" (critical, more on that below!)
  • Created a draft of my first blog post, just to make sure everything rendered properly

Step 3: First Push‚Ä¶ to the Wrong Repo

Here‚Äôs where things got entertaining. I initially pushed to a project repo named something like https://cmutter79.github.io/cmutter79 instead of the proper cmutter79.github.io. Result? The site appeared ‚Äî and looked like it was from 1996. Oops.

Step 4: The Great baseurl Lesson

If your site is hosted at a subdirectory like /my-site, you need to set baseurl: "/my-site" in your config. But since I was using a user site, I needed baseurl: "".

Troubleshooting Git Like a Pro(duct Manager)

At one point I ran into this gem of an error:


error: failed to push some refs to 'https://github.com/cmutter79/repo-name.git'
hint: Updates were rejected because the remote contains work that you do not have locally.

Turns out I needed to either pull remote changes or force push if I wanted my local version to overwrite. Since I knew I hadn‚Äôt added anything important to GitHub yet, I used:
