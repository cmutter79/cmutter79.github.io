Exploring Agentic AI Through a Job Application Automation Project

Roughly a month ago, I was unexpectedly laid off from my role in product management. Definitely not ideal, but it's the unfortunate reality many of us have faced recently so I'm trying to make the best of it. Since then, I’ve been navigating the job market and reflecting on how much the application landscape has evolved.

Positions fill quickly. Some postings receive hundreds of applicants within the first day. Generic resumes no longer make the cut. Tailoring each application is now essential. While AI tools are available, much of the effort still requires manual work: interpreting job postings, drafting cover letters, updating resumes, and managing all of it on a per-role basis.

In parallel, there is an increasing demand for AI literacy in Product Management roles. In fact, during a training session at my previous company, the message was clear: if a company doesn't have an AI strategy, it's already falling behind. That same urgency applies to job seekers.

So I asked myself: could I create a tool that supports my job search while also helping me sharpen my AI and technical skills?

Motivation for Building the Tool

This project began as a solution to a repetitive process:

  • Creating a folder for each new job opportunity
  • Copying my master resume
  • Using ChatGPT to draft a custom cover letter
  • Asking ChatGPT for feedback to improve my resume
  • Editing manually, then applying

This workflow, while helpful, was time-consuming and fragmented. I wanted to consolidate and streamline the process with automation.

Current Capabilities

Job Summary and Fit Analysis

The tool takes a job posting URL, parses its content, and produces a summary that highlights required skills. It also evaluates how well my existing resume aligns with the role, identifying strengths and opportunities for improvement.

Draft Cover Letter

Based on the job description and my resume, the tool generates a personalized draft cover letter. This provides a solid starting point that I can refine further.

Resume Suggestions

The system analyzes the job description and suggests specific changes to my resume to improve its alignment with the role. While I still manually update the resume, these insights are helpful.

Privacy-First Design Using Ollama

I chose to run everything locally using Ollama. This setup protects my data and allows me to test different open-source models. Although the performance is slower compared to cloud solutions, the tradeoff is worth it.

System Architecture and Components

Inputs

  • A job posting URL entered by the user
  • A saved copy of my master resume

Job Parsing Agent

Scrapes and cleans the job posting text using Langchain’s webBaseLoader. This process converts the listing into a format suitable for analysis.

Summarization Agent

Creates a concise summary of the job posting to make it easier to evaluate.

Skills Extraction Agent

Identifies and lists the key skills and qualifications mentioned in the job description.

Fit Analysis Agent

Compares the posting against my resume. It outlines areas where I match well, areas where I do not, and suggestions for better alignment.

Job Summary Document

Generates a Microsoft Word document that includes the summary, fit analysis, and original job posting content.

Cover Letter Agent

Generates a draft cover letter using the job summary and my resume. The result is stored as a Word document.

Resume Enhancement Agent

Produces a list of resume modifications to improve alignment with the job.

Technical Stack

  • Language Model: Ollama (tested with gemma3:4b and llama3.2:latest)
  • Programming Language: Python
  • Document Generation: python-docx
  • File Operations: Standard Python OS and JSON libraries
  • Interface: Command-line interface

Future Enhancements

  • Fine-tune prompts to improve agent performance
  • Build a lightweight web interface using Flask
  • Implement a job fit scoring system
  • Evaluate job benefits and compensation as part of fit analysis
  • Reanalyze resume fit after changes
  • Track applications and follow-ups
  • Store previous applications in a database for pattern matching
  • Add an interview preparation component

Lessons Learned

Rapid Development

With the assistance of ChatGPT, I built the initial prototype in about six hours. This accelerated pace made development enjoyable and productive.

Incremental Progress

Instead of building an all-in-one platform from the start, I focused on solving one problem at a time. This approach simplified debugging and kept the project manageable.

Learning by Building

This project has helped me understand how agentic workflows can be structured using LLMs. It has also sparked several new ideas for future tools.

Practical Power of Agents

Even in a basic setup, these agents are capable of handling unstructured data and generating useful, context-aware content. Just a few years ago, building something like this would have required a team and significant time. Today, it’s possible with a laptop and a bit of curiosity.
